# platform-shell

React platform shell that orchestrates micro-frontends with authentication and multi-tenant organization switching.

## What it generates

A host application using Vite Module Federation that:
- Authenticates users via **Clerk** (default) or **generic OIDC** (Keycloak, Auth0, Azure AD)
- Provides an **organization/tenant switcher** in the header
- Loads **micro-frontends** (generated by `frontend-app`) as sidebar menu items
- Uses a **runtime `remotes.json`** config — add/remove micro-frontends without rebuilding
- Includes a shared `useApi()` hook that automatically attaches Bearer tokens and `X-Tenant-ID` headers

## Architecture

```
┌──────────────────────────────────────────────┐
│ Header: [Logo] [Org Switcher ▾] [User ▾]    │

```

## Usage

```bash
# 1. Generate the shell
appgen generate platform-shell -o ./my-platform \
  -s projectName=my-platform \
  -s clerkPublishableKey=pk_test_xxxxx

# 2. Add frontend apps (--shell auto-registers them)
appgen generate frontend-app -o ./orders -s projectName=orders -s devPort=5001 --shell ./my-platform
appgen generate frontend-app -o ./users -s projectName=users -s devPort=5002 --shell ./my-platform
```

Each `--shell` call appends to `my-platform/public/remotes.json`. The shell reads this at startup — no rebuild needed.

```bash
# With generic OIDC instead of Clerk
appgen generate platform-shell -o ./my-platform \
  -s projectName=my-platform \
  -s authProvider=oidc \
  -s oidcAuthority=https://keycloak.example.com/realms/main \
  -s oidcClientId=my-platform
```

## How remote apps are configured

The shell reads `public/remotes.json` at startup:

```json
[
  { "name": "orders", "url": "http://localhost:5001", "menuLabel": "Orders", "menuIcon": "" },
  { "name": "users", "url": "http://localhost:5002", "menuLabel": "Users", "menuIcon": "" }
]
```

This file is:
- **Written automatically** by `appgen generate frontend-app --shell`
- **Editable manually** — add, remove, or reorder apps without regenerating
- **Environment-specific** — swap URLs for staging/prod without rebuilding
- **Read at startup** by `src/config/remotes.ts` via `loadRemotes()`

The sidebar, routes, and Module Federation remotes all derive from this one file.

## Parameters

| Parameter | Default | Description |
|-----------|---------|-------------|
| `projectName` **(required)** | -- | Project name (kebab-case) |
| `projectTitle` | derived | Display name |
| `authProvider` | `clerk` | `clerk` or `oidc` |
| `clerkPublishableKey` | `pk_test_REPLACE_ME` | Clerk publishable key (clerk only) |
| `oidcAuthority` | `https://auth.example.com/realms/main` | OIDC issuer URL (oidc only) |
| `oidcClientId` | `platform-shell` | OAuth2 client ID (oidc only) |
| `oidcScopes` | `openid profile email` | OAuth2 scopes (oidc only) |
| `apiBaseUrl` | `/api` | Backend API base URL |
| `tenantsEndpoint` | `/api/tenants` | Tenant list endpoint (oidc only) |
| `nodeVersion` | `20` | Node.js version |
| `packageManager` | `pnpm` | npm, pnpm, or yarn |
| `containerRegistry` | `ghcr.io` | Container registry |
| `containerRegistryOrg` | `your-org` | Registry organization |
| `k8sNamespace` | `default` | Kubernetes namespace |

## Feature flags

| Feature | Default | Description |
|---------|---------|-------------|
| `docker` | on | Dockerfile + nginx config |
| `kubernetes` | on | Kustomize base + dev/prod overlays |
| `cicd` | on | GitHub Actions (CI, build-push, deploy) |
| `tailwind` | on | Tailwind CSS + PostCSS |

Disable with: `-s features.docker=false`

## Auth providers comparison

| | Clerk | OIDC |
|---|---|---|
| Setup | Sign up at clerk.com, get key | Deploy Keycloak/Auth0, configure realm |
| Login UI | Hosted modal (`<SignInButton>`) | Redirect to provider |
| Org switcher | Built-in `<OrganizationSwitcher>` | Custom dropdown + API |
| User management | Clerk dashboard | Provider admin console |
| Token format | JWT (Clerk-issued) | JWT (provider-issued) |
| Dependencies | `@clerk/clerk-react` | `oidc-client-ts` + `react-oidc-context` |
